    Introduction

    You can get an API key from our developer page.

    Your API token is very important, DO NOT SHARE WITH OTHERS. Others can access your Dynalist data with your API key, so be very careful.
    In the example returned JSON responses, placeholder text like "<document id>" and "<node id>" will be replaced by real values.

    Documents and folders

    Get all documents and folders

    This endpoint returns JSON structured like this:

    [
  {
    "id": "<document id>",
    "title": "Todo list",
    "type": "document",
    "permission": 4
  },
  {
    "id": "<folder id>",
    "title": "Study notes",
    "type": "folder",
    "permission": 2,
    "collapsed": false,
    "children": [
      {
        "id": "<document id>",
        "title": "Psychology 101",
        "type": "document",
        "permission": 2
      }
    ]
  }
]

    This endpoint fetches all your documents and folders.

    HTTP Request

    GET http://dynalist.io/api/v1/file/list

    Query Parameters

    Parameter Type Description
    token String Your API key.

    In the returned response, the "permission" enumerate has the following possible values:

    Value Meaning
    0 No access
    1 Read only
    2 Edit rights
    3 Manage
    4 Owner

    Make changes to documents and folders

    Example request:

    [
  {
    "action": "edit",
    "type": "document",
    "file_id": "<document id>",
    "title": "New title"
  },
  {
    "action": "move",
    "type": "folder",
    "file_id": "<folder id>",
    "parent_id": "<folder id>",
    "index": 3
  }
]

    This endpoint returns JSON structured like this:

    [
  true,
  false
]

    This endpoint lets you make changes to specific documents and folders.

    HTTP Request

    GET http://dynalist.io/api/v1/file/edit

    URL parameters

    Parameter Type Description
    token String Your API key.
    changes Array Array of changes to perform (see below for the required format).

    Change format

    Parameter Type Description
    action String Action to perform. Must be either move or edit. Use move to move the document or folder under another folder, or use edit to change the title of the document or folder.
    type String Type of the target file. Must be either folder or document.
    file_id String The ID of the document or folder you're trying to change or move.
    parent_id String Required if action is move. The file ID of the parent folder you want to move the target file to.
    index Integer Required if action is move. This field is an integer that's the zero-indexed position you want the file to land in the parent folder, must be represent if action is move, supply -1 to place it at the end, and 0 to place it at the top.
    title String Required if action is edit. The new title that you're trying to set.

    This endpoint fetches the content of a specific document.

    HTTP Request

    DELETE http://dynalist.io/api/v1/doc/read

    URL Parameters

    Parameter Type Description

    Make change to the content of a document

    Example request:

    [
  {
    "action": "insert",
    "parent_id": "<parent node id>",
    "index": 0,
    "content": "Read the second book in the series",
    "note": "Return the book to the library by 04/23."
    "checked": false
  },
  {
    "action": "edit",
    "node_id": "<target node id>",
    "checked": true
  },
  {
    "action": "move",
    "node_id": "<target item ID>",
    "parent_id": "<parent item ID>",
    "index": 2
  },
  {
    "action": "delete",
    "node_id": "<target item ID>"
  }
]

    This endpoint returns JSON structured like this:

    [
  true,
  false
  true,
  true
]

    This endpoint lets you make changes to the content of a specific document.

    HTTP Request

    GET http://dynalist.io/api/v1/doc/edit

    URL parameters

    Parameter Type Description
    token String Your API key.
    file_id String The ID of the document you're trying to change.
    changes Array Array of changes to perform (see below for the required format).

    Change format

    Parameter Type Description
    action String Action to perform. Must be one of insert, edit, move or delete.
    node_id String Required if action is edit, move, or delete. The ID of the item to change.
    parent_id String Required if action is insert or move. The ID of the parent item
    index Integer Required if action is move. This field is an integer that's the zero-indexed position you want the file to land in the parent folder, must be represent if action is move, supply -1 to place it at the end, and 0 to place it at the top.
    content String New item title. Used for the insert and edit actions.
    note String New item note. Used for the insert and edit actions.
    checked Boolean New item checked status. Used for the insert and edit actions.

    Send to inbox

    This endpoint returns JSON structured like this:

    true

    This endpoint lets you make changes to the content of a specific document.

    HTTP Request

    GET http://dynalist.io/api/v1/inbox/add

    URL parameters

    Parameter Type Description
    token String Your API key.
    index Integer Zero-indexed position you want the file to land in the inbox item. Use -1 to place it at the end, and 0 to place it at the top. If not given, the user setting will be used (default to bottom).
    content String Title of the new item. Default: empty.
    note String Note of the new item. Default: empty.
    checked Boolean Checked status of the new item. Default: false.
